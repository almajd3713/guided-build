# Contributing

Contributions should improve transfer across projects, evidence quality, safety, privacy, or workflow efficiency.

Before proposing a workflow rule:

1. Identify a repeated failure in a real or fixture evaluation.
2. Add or update an evaluation that reproduces it.
3. Put deterministic constraints in the CLI when practical.
4. Keep skill prose focused on decisions that require agent judgment.
5. Verify that unrelated fixture projects do not inherit domain-specific language.

Run the repository tests and Codex skill/plugin validators. Do not include private learning-state exports in issues, fixtures, or pull requests.
